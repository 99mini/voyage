인터넷 네트워크

## IP (Internet Protocol)

### 역할

- 지정한 IP 주소에 데이터 잔달
- 패킷(packet) 단위로 데이터 전송

### 한계

- 비연결성
  - 패킷을 받을 대상이 없거나 서비스 불능 상태여도 패킷 전송
- 비신뢰성
  - 패킷 손실, 중복, 순서 변경 가능성
- 프로그램 구분
  - IP 주소만으로는 어떤 프로그램에 데이터를 전달할지 알 수 없음: 같은 IP를 사용하는 서버에서 통신하는 애플리케이션이 둘 이상일 경우

## TCP/UDP

:::info 인터넷 프로토콜 계층

| 계층                     | 프로토콜 예시   | 역할                                       |
| ------------------------ | --------------- | ------------------------------------------ |
| 애플리케이션 계층        | HTTP, FTP, SMTP | 애플리케이션 간 통신                       |
| 전송 계층                | TCP, UDP        | 호스트 간 데이터 전송 및 신뢰성 보장       |
| 인터넷 계층              | IP              | 패킷 전달 및 라우팅                        |
| 네트워크 인터페이스 계층 | Ethernet, Wi-Fi | 물리적 네트워크 연결 및 데이터 프레임 전송 |

:::

### TCP

전송 제어 프로토콜 (Transmission Control Protocol)

- 연결 지향적 프로토콜 - TCP 3 way handshake (가상 연결)
- 데이터 전달 보증
- 순서 보장

#### 3 way handshake

1. 클라이언트가 서버에 `SYN` 패킷 전송 (연결 요청)
2. 서버가 클라이언트에 `SYN-ACK` 패킷 전송 (연결 수락 및 응답)
3. 클라이언트가 서버에 `ACK` 패킷 전송 (응답 확인)

#### 순서 보장

- 각 패킷에 시퀀스 번호 부여
- 수신 측에서 시퀀스 번호를 기반으로 패킷 재조립

### UDP

사용자 데이터그램 프로토콜 (User Datagram Protocol)

- 비연결형 프로토콜
- 데이터 전달 보증 없음
- 순서 보장 없음
- Port 번호를 사용하여 프로그램 구분
  - 체크썸 검사로 데이터 무결성 확인 가능
  - 애플리케이션 계층에서 추가 작업 필요

## PORT

IP 주소 내에서 특정 프로세스(프로그램)를 식별하는 논리적 번호

- 0 ~ 65535까지 존재
- 0 ~ 1023: 잘 알려진 포트 (Well-known Ports)
  - HTTP: 80
  - HTTPS: 443
  - FTP: 20, 21
  - Telnet: 23

## DNS (Domain Name System)

도메인 이름을 IP 주소로 변환하는 시스템

```text
----------                           -------------
| client | -- 1.example.com 요청 --> | DNS 서버    |
----------                           -------------
  |  ^                                    |
  |  |-- 2. example.com의 100.100.100.2 ---|
  |
  |
  |------- 3. 접속 100.100.100.2 ----> -----------
                                     |  웹 서버   |
                                      -----------
```
