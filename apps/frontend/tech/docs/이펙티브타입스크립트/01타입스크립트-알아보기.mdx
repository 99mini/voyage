## 아이템 1: 타입스크립트와 자바스크립트의 관계 이해하기

> 작성일: 2025-02-25T13:55

- 타입스크립트는 자바스크립트의 상위집합(superset)
  - 모든 자바스크립트 프로그램은 이미 타입스크립트 프로그램
- 자바스크립트의 런타임 동작을 모델링
  - 단, 타입 체커를 통과했다고 해서 런타임 오류가 항상 발생하지 않는 것은 아니다.

```typescript title="런타임 오류가 될 만한 코드 (.ts)"
const a = null + 7; // 자바스크립트에서는 a 값이 7이 된다.
//        ~~~~~~~~ Operator '+' cannot be applied to types 'null' and '7'
const b = [] + 12; // 자바스크립트에서는 b 값이 '12'이 된다.
//        ~~~~~~~  Operator '+' cannot be applied to types 'undefined[]' and 'number'.
alert('Hello', 'TypeScript'); // 자바 스크립트에서는 alert 값이 'Hello'이 된다.
//              ~~~~~~~~~~~~ Expected 0-1 arguments, but got 2
```

## 아이템 2: 타입스크립트의 설정 이해하기

> 작성일: 2025-02-25T23:45

- `tsconfig.json`을 사용하여 타입스크립트 설정을할 수 있다.

```typescript title="tsconfig.json"
{
  "compilerOptions": {
    "target": "es5", // ES5로 타입스크립트 코드를 컨파일.
    "module": "commonjs", // CommonJS 모듈로 타입스크립트 코드를 컨파일.
    "noImplicitAny": true, // any로 타입을 추론하는 것을 방지.
    "strictNullChecks": true, // null와 undefined 런타임 오류를 발생시큰로 추측. `"undefined는 객체가 아닙니다."`와 같은 런타임 오류 방지
    "strict": true, // 타입스크립트에서 업격한 체크
  }
}
```

**noImplicitAny**

```typescript title="noImplicitAny: false"
function add(a, b) {
  // 타입 체커 통과!
  return a + b;
}
```

`add` 함수에 마우스를 올려 보면, 타입스크립트가 추론한 타입을 알 수 있다. (모두 `any`타입으로 추론)

```typescript title="ts"
function add(a: any, b: any): any;
```

**strictNullChecks**

```typescript title="strictNullChecks: false"
const x: number = null; // 정상, null은 유효한 값.
```

```typescript title="strictNullChecks: true"
const x: number = null;
//    ~ Type 'null' is not assignable to type 'number'.ts(2322)
```
