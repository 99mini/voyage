# 꿈을 기록하고 해석하기(Dream Interpretation) PRD

> 작성일: 2025년 02월 27일  
> 최종수정일: 2025년 02월 27일

## 1. 개요

이 문서는 꿈을 기록하고 AI 기반 해몽을 제공하는 애플리케이션의 요구사항을 정의합니다. 사용자가 꿈을 쉽게 기록하고, AI를 활용하여 해몽을 제공하는 기능을 포함합니다.

### **발상 및 가설**

1. 꿈을 꾸는 도중에 잠을 깨면 일시적으로 꿈 내용이 기억나지만, 곧 잊어버린다.
2. 사용자들은 기억에 남기고 싶은 꿈을 기록하고 싶어 한다.
3. 꿈을 기록하는 것을 넘어 AI를 활용한 해몽 서비스에 대한 니즈가 있다.

---

## 2. 기술 요구사항

- **모바일 앱을 우선 개발**하며, 웹에서도 접근할 수 있도록 웹뷰(WebView) 기반으로 제공
- **회원가입 없이도 이용 가능**하며, 회원가입을 하면 기록을 보존할 수 있도록 함
- **AI 기반 해몽 서비스 제공** (스트리밍 방식 구현 필요)
- **클라이언트와 서버 간의 원활한 데이터 통신**

### **FE (프론트엔드)**

- **모바일 앱 우선 개발** (React Native, Expo 활용)
- **웹뷰 및 네이티브 앱 통신 구현** (Remix.js 기반 웹 애플리케이션)
- **푸시 알림 기능 추가** (기상 시간에 맞춘 알림 제공)

### **BE (백엔드)**

- **AI 요청 스트리밍 구현** (서버에서 실시간 해몽 결과 반환)
- **회원가입 및 꿈 기록 아카이빙을 위한 데이터베이스 설계**
- **사용자 맞춤형 추천 해몽 제공** (기록된 꿈 패턴 분석)

---

## 3. 기능 요구사항

### **3.1 꿈 기록하기**

- 사용자가 설정한 시간(기상 시간)에 푸시 알림을 보냄
- 사용자가 꿈을 기록할 수 있는 인터페이스 제공 (텍스트 기반 입력)
- 꿈을 태그 또는 카테고리로 분류할 수 있도록 지원

### **3.2 AI 기반 해몽 서비스**

- 사용자가 기록한 꿈 중에서 원하는 항목에 대해 AI 해몽 요청 가능
- AI가 해몽을 수행하여 사용자에게 반환 (스트리밍 처리)
- 해몽 결과를 푸시 알림을 통해 제공
- 사용자 피드백(유사한 해몽 추천, 해몽 평가 기능) 반영 가능

### **3.3 회원가입 및 로그인**

#### **비회원**

- 꿈 기록 가능 (로컬 저장)
- AI 해몽 서비스 이용 가능 (주 X회 제한)

#### **일반 회원**

- 꿈 기록 가능 (클라우드 저장)
- AI 해몽 서비스 이용 가능 (제한 있음)
- 꿈과 해몽을 아카이빙 가능

#### **구독 회원**

- 꿈 기록 가능 (클라우드 저장)
- AI 해몽 서비스 무제한 이용 가능
- 꿈과 해몽을 아카이빙 가능
- 개인 맞춤형 해몽 및 심층 분석 제공
- 기타 프리미엄 기능 (예: 꿈 패턴 분석, 데이터 시각화)

### **3.4 해몽 아카이빙**

- 회원가입한 사용자만 꿈과 해몽을 아카이빙 가능
- 꿈 기록을 기반으로 꿈 패턴을 분석하는 기능 제공 (예: 반복되는 꿈 경향 분석)

### **3.5 해몽 커뮤니티 (추가 고려 기능)**

- 사용자가 자신의 꿈과 해몽을 공유할 수 있는 공간 제공
- 다른 사용자의 꿈과 해몽에 대해 피드백 및 공감 가능
- 인기 해몽 및 추천 해몽 제공

---

## 4. 비즈니스 모델

### **4.1 광고 수익**

- 무료 사용자를 대상으로 광고 노출
- 해몽 페이지 및 결과 화면에서 광고 배너 제공

### **4.2 구독 서비스**

- 월간 또는 연간 정액제 구독 모델 제공
- 구독 회원에게 추가적인 해몽 분석 및 맞춤형 추천 기능 제공

---

## 5. 시스템 아키텍처

### **5.1 프론트엔드 (FE)**

- **웹뷰 기반 앱** (Remix.js + React Native)
- **푸시 알림 연동** (Expo Notifications 활용)
- **로컬 데이터 저장 및 동기화 지원** (AsyncStorage, SecureStore 등 활용)

### **5.2 백엔드 (BE)**

- **AI 해몽 요청 스트리밍 처리** (WebSocket 또는 SSE 적용)
- **클라이언트 요청 스트리밍 지원**
- **사용자 데이터 저장 (NoSQL / PostgreSQL 기반)**

### **5.3 데이터 흐름**

1. 사용자가 꿈을 기록하고 저장 (로컬 또는 클라우드)
2. AI 해몽 요청 시, 백엔드에서 처리 후 실시간 스트리밍으로 반환
3. 회원 가입된 사용자에게 데이터 아카이빙 및 분석 기능 제공
4. 구독 회원에게 맞춤형 해몽 분석 및 추천 제공
